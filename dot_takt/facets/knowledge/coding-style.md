# Remotion 実装コーディングスタイル

## TypeScript
- strict modeを有効にする
- 型推論に頼りすぎず、明示的な型定義を優先する

## ESLint
- プロジェクトのESLint設定に従う
- 警告を残さない

## Prettier
- 統一されたフォーマットを使用する
- 設定ファイルに従う

## Component設計
- 関数コンポーネントのみ
- Propsはインターフェースで定義する
- デフォルト値は明示的に指定する

## 型定義
- anyの使用を禁止する
- 共通型はtypes/に配置する
- 動画固有の型はvideos/<id>/data/に配置する

## 共通化方針
- 2つ以上の動画で使う可能性があるロジックは共通化する
- マジックナンバーを散在させない（定数として定義）
- フレームレートを各所に固定値として埋め込まない

## コンポーネント設計
- 表示とロジックを分離する
- 動画固有のデータはpropsで受け取る
- 状態を持たないコンポーネントを優先する
