# Retrospective Report
次の構造で報告してください。

## Process Summary
制作プロセス全体の概要と所要時間。

## Bottlenecks
特定されたボトルネックとその原因。

## Knowledge Candidates
抽出された知識候補。各候補について以下を記載。
- Category: Knowledge / Persona / Policy / Instruction / Output Contract
- Content: 具体的な内容
- Target: 適用先（ファイルパス）

## Template Proposals
テンプレート更新の具体的な提案一覧。

## Adoption Priority
導入優先順位（High / Medium / Low）とその理由。

## Status
最後の行に、次のいずれか一つだけを記載する。
REVIEW_APPROVED
REVIEW_NEEDS_FIX
