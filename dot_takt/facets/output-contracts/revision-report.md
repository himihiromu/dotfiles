# Revision Report
次の構造で報告してください。

## Summary
修正の概要と対象成果物。

## Changes
各変更について以下を記載。
- Before: 変更前の内容
- After: 変更後の内容
- Reason: 変更理由
- Expected Improvement: 期待される改善効果

## Validation
修正後に実施した確認事項。未実施事項も明記。

## Remaining Issues
この修正サイクルで対応しきれなかった課題。なければ「なし」。

## Status
最後の行に、次のいずれか一つだけを記載する。
STEP_COMPLETE
STEP_BLOCKED
